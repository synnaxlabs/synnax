The HTTP device driver allows the Synnax Driver to connect to REST/HTTP API endpoints
for data acquisition and control. It supports polling JSON APIs at configurable rates
and sending commands via HTTP requests.

<Divider.Divider x />

## Use Cases

The HTTP driver is designed for integrating with any system that exposes an HTTP/REST
API:

- IoT platforms and gateways
- REST APIs and web services
- Cloud data sources and monitoring endpoints
- Custom HTTP endpoints for sensor data
- Industrial systems with HTTP-based interfaces

<Divider.Divider x />

## Supported Protocols

**The Synnax Driver supports both HTTP and HTTPS.** SSL certificate verification can be
toggled per connection, allowing integration with self-signed certificates in
development or internal environments.

<Note.Note variant="info" className="compact">
  Set `verify_ssl` to `false` when connecting to servers with self-signed certificates.
  This is not recommended for production environments.
</Note.Note>
