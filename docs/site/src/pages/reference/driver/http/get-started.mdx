The HTTP device driver allows the Synnax Driver to communicate with any device or
service that exposes an HTTP or HTTPS API. This makes it possible to acquire data from
REST endpoints, send commands to web services, and integrate with custom HTTP servers
that don't use a traditional industrial protocol.

<Divider.Divider x />

## Supported Protocol

**The Synnax Driver supports HTTP and HTTPS**, enabling communication with any server
that exposes a JSON-based REST API. The HTTP driver is commonly used for:

- Custom REST APIs on embedded devices and microcontrollers
- Web services and cloud APIs that expose telemetry data
- Internal monitoring endpoints (health checks, metrics)
- IoT gateways and edge devices with HTTP interfaces
- Devices that don't support traditional industrial protocols (OPC UA, Modbus, etc.)

<Note.Note variant="info" className="compact">
  The HTTP driver extracts values from JSON response bodies using [JSON
  Pointers](https://datatracker.ietf.org/doc/html/rfc6901) (RFC 6901), and supports
  Bearer, Basic, and API Key authentication.
</Note.Note>

<Divider.Divider x />

## Supported HTTP Methods

| Method    | Description                    |
| --------- | ------------------------------ |
| `GET`     | Retrieve data from a server    |
| `POST`    | Send data to a server          |
| `PUT`     | Replace a resource on a server |
| `PATCH`   | Partially update a resource    |
| `DELETE`  | Delete a resource on a server  |
| `HEAD`    | Retrieve headers only          |
| `OPTIONS` | Retrieve supported methods     |
