The Synnax Driver supports hardware from a variety of vendors, such as
[National Instruments](https://ni.com), [LabJack](https://labjack.com), and
[EtherCAT](https://ethercat.org) devices. The Driver also supports acquiring data from
servers via HTTP JSON REST APIs, [Modbus TCP/IP](https://modbus.org), and
[OPC UA](https://opcfoundation.org). The Driver also supports sending alerts to
[PagerDuty](https://www.pagerduty.com/) when statuses change. Finally, the Driver
supports running [Arc automations](/reference/control/arc/introduction), Synnax's
automation language for hardware control. Synnax also supports building custom drivers
for devices that are not natively supported.

## Synnax Driver

The Driver natively supports acquiring data from and sending commands to
[National Instruments](/reference/driver/ni/get-started),
[EtherCAT](/reference/driver/ethercat/get-started), and
[LabJack](/reference/driver/labjack/get-started) hardware; connecting to
[HTTP](/reference/driver/http/get-started),
[OPC UA](/reference/driver/opc-ua/connect-server), and
[Modbus](/reference/driver/modbus/get-started) servers; sending alerts to
[PagerDuty](/reference/driver/pagerduty/get-started); and running
[Arc automations](/reference/control/arc/introduction).

### Embedded Driver

Every instance of the [Synnax Core](/reference/core/quick-start) comes with the Driver
embedded inside. This Driver will run on the same machine as the Core, and starts
automatically when the Core starts. The embedded Driver is recommended for most users.

### Driver Installation

The [Driver](/reference/driver/installation) can also be installed as a separate binary
that can run on a different machine from the Core. It is used to integrate data
acquisition devices across multiple machines. The Driver connects to a Core over the
network.

#### Support for NI Linux Real-Time

The Driver supports deployment on
[NI Linux Real-Time](https://www.ni.com/en/shop/linux/introduction-to-ni-linux-real-time.html)
controllers, such as [NI CompactRIO](https://www.ni.com/en/shop/compactrio.html) and
[NI PXI](https://www.ni.com/en/shop/pxi.html) systems.

In addition to performing data acquisition and control tasks on real-time controllers,
the Driver also supports running [Arc automations](/reference/control/arc/introduction)
on the real-time controller, allowing for precisely timed control loops that run close
to the hardware without introducing latency from network communication.

## Building a Custom Driver

If you don't see a driver for your hardware, you can build your own using the
[Python client](/reference/python-client/device-driver).
