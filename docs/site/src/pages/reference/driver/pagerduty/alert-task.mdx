## Task Configuration Reference

| Parameter     | Type    | Required | Default | Description                                                    |
| ------------- | ------- | -------- | ------- | -------------------------------------------------------------- |
| `name`        | string  | Yes      | -       | Human-readable task name                                       |
| `routing_key` | string  | Yes      | -       | 32-character PagerDuty Events API v2 Integration Key           |
| `auto_start`  | boolean | No       | `false` | Automatically start the task after configuration               |
| `alerts`      | array   | Yes      | -       | List of [alert configurations](#alert-configuration-reference) |

<Divider.Divider x />

## Alert Configuration Reference

Each alert maps a Synnax status key to a PagerDuty event. At least one alert must be
enabled.

| Parameter                 | Type    | Required | Default | Description                                                           |
| ------------------------- | ------- | -------- | ------- | --------------------------------------------------------------------- |
| `status`                  | string  | Yes      | -       | The Synnax status key to watch                                        |
| `enabled`                 | boolean | No       | `true`  | Whether this alert is active                                          |
| `treat_error_as_critical` | boolean | No       | `false` | Map error variant to `critical` PagerDuty severity instead of `error` |
| `component`               | string  | No       | `""`    | Component of the source machine (e.g., `"engine-1"`)                  |
| `group`                   | string  | No       | `""`    | Logical grouping of components (e.g., `"engines"`)                    |
| `class`                   | string  | No       | `""`    | Class/type of the event (e.g., `"engine-failure"`)                    |

## How-To

<Client.Tabs client:load exclude={["cpp", "typescript"]}>
  <Fragment slot="console">

Open the Search and Command Palette with `Ctrl+Shift+P` (Windows) or `Cmd+Shift+P`
(macOS). In command mode (first character is `>`), type "PagerDuty" and select **Create
PagerDuty Alert Task**.

In the task configuration form:

1. Enter your 32-character PagerDuty **routing key**
2. Add one or more alerts by clicking the add button
3. For each alert, select the **status** to watch
4. Optionally configure **component**, **group**, and **class** metadata
5. Click **Configure** to save, then press play to start the task

<Text.Text slot="summary" level="p"><Icon.Play /> Create and run an alert task</Text.Text>

```python
import synnax as sy

client = sy.Synnax()

# Create a status to watch
STATUS_KEY = "example-pagerduty-status"
client.statuses.set(
    sy.Status(
        key=STATUS_KEY,
        name="Engine 1",
        variant="success",
        message="Engine 1 is healthy",
    )
)

# Create the PagerDuty alert task
alert_task = sy.pagerduty.AlertTask(
    name="Example PagerDuty Alert",
    routing_key="<your-32-character-integration-key>",
    auto_start=False,
    alerts=[
        sy.pagerduty.AlertConfig(
            status=STATUS_KEY,
            enabled=True,
            treat_error_as_critical=True,
            component="engine-1",
            group="engines",
        ),
    ],
)

# Configure the task on the server
client.tasks.configure(alert_task)

# Start the task - it will watch for status changes
with alert_task.run():
    # Trigger a PagerDuty incident by setting status to error
    client.statuses.set(
        sy.Status(
            key=STATUS_KEY,
            name="Engine 1",
            variant="error",
            message="Engine 1 is down",
            description="Connection to database lost",
        )
    )
    sy.sleep(5)

    # Resolve by setting status back to success
    client.statuses.set(
        sy.Status(
            key=STATUS_KEY,
            name="Engine 1",
            variant="success",
            message="Engine 1 recovered",
        )
    )
    sy.sleep(3)
# Task automatically stops when exiting the context manager
```
