This guide covers building custom device drivers in Python to integrate hardware that isn't natively supported by Synnax's built-in drivers. This is useful for prototyping or integrating specialized hardware.

**Python-specific feature**: Device driver development is currently only supported in Python. For production-grade drivers, see the C++ Driver documentation.

## Production Drivers

<Note.Note variant="info">
  **For production-grade C++ driver development**: See the <a href="/reference/driver/">Driver documentation</a> for building robust, high-performance hardware integrations with built-in support for:
  - **LabJack** devices (U3, U6, T4, T7, T8)
  - **National Instruments** DAQmx devices
  - **OPC UA** servers
  - **Modbus TCP/IP** devices

  C++ drivers offer better performance, built-in device pooling, cross-platform support, and integration with the Synnax task system.
</Note.Note>
