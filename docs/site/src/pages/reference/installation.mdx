This page will walk you through installing and running Synnax locally. By the end,
you'll have a running Core and a Console connected to it. Select your operating system
to get started.

<Divider.Divider x />

## <Step step={1} level="h2">Install the Core</Step>

The Core is the heart of every Synnax deployment. It stores all sensor and actuator data
and serves as a central hub where devices, Consoles, and control sequences communicate.

<Platform.Tabs client:load>

<Fragment slot="Docker">

### Docker

<Note.Note variant="warning">
  A Synnax Core within a container has an isolated runtime, which means that it can't
  access devices attached to the host machine. If you plan to connect data acquisition
  devices using the Synnax Driver, install Synnax directly on the host OS using one of
  the other methods.
</Note.Note>

Pull the latest image from Docker Hub:

```bash
docker pull synnaxlabs/synnax:latest
```

</Fragment>

<Fragment slot="Linux">

### Linux

Download the latest binary:

<DownloadURL os="linux" />

Move the binary to `/usr/local/bin` and make it executable:

<MoveCommand os="linux" />

```bash
chmod +x /usr/local/bin/synnax
```

If `/usr/local/bin` is not in your `PATH`, add it permanently:

```bash
echo 'export PATH=$PATH:/usr/local/bin' >> ~/.bashrc
```

For `zsh`, replace `~/.bashrc` with `~/.zshrc`.

Verify the installation:

```bash
synnax --version
```

You should see:

<VersionOutput />

</Fragment>

## <Step step={2} level="h2">Start the Core</Step>

<Platform.Tabs client:load>

<Fragment slot="Docker">

```bash
docker run -p 9090:9090 synnaxlabs/synnax --listen=localhost:9090 --mem --insecure
```

</Fragment>

<Fragment slot="Linux">

```bash
synnax start --listen=localhost:9090 --mem --insecure
```

</Fragment>

<Fragment slot="macOS">

```bash
synnax start --listen=localhost:9090 --mem --insecure
```

</Fragment>

<Fragment slot="Windows">

```powershell
synnax start --listen=localhost:9090 --mem --insecure
```

</Fragment>

</Platform.Tabs>

This starts a Core with the following configuration:

<table>
  <thead>
    <tr>
      <th>Flag</th>
      <th>Description</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>`--listen=localhost:9090`</td>
      <td>Sets the address the Core will listen on</td>
    </tr>
    <tr>
      <td>`--mem`</td>
      <td>Store all data in memory (useful for learning and development)</td>
    </tr>
    <tr>
      <td>`--insecure`</td>
      <td>Run without TLS</td>
    </tr>
  </tbody>
</table>

The `--mem` flag is great for getting started, but data will be lost when the Core
stops. See the [CLI Reference](/reference/core/cli-reference) for persistent storage
options and other configuration flags. For production deployments, see
[Production](/reference/core/production).

To stop the Core, press `Ctrl+C` in the terminal.

## <Step step={4} level="h2">Connect the Console to the Core</Step>

The Console ships with a pre-configured connection to a local Core. It should appear in
the list of available cors on the login page as 'Local'. If the core you started in the
previous step is reachable, the status badge will turn blue and say 'Connected'.
