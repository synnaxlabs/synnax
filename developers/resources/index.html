Resources

The following is a collection of useful resources for developing data intensive distributed systems. They are organized by category, but some resources may be relevant to multiple categories.

Data Processing​

Martin Kleppmann - Designing Data-Intensive Applications​

Link: Designing Data-Intensive Applications

A one thousand page behemoth that peers under the hood of large scale data-processing systems like Linux, Spark, Hadoop + HDFS, GFS, SQL, NoSQL. It introduces some of the fundamental concepts of distributed systems, like ACID Compliance, CAP Theorem, and Serializability. It also covers some of the history behind the significant players in the space, and how the solutions have changed over time.

Data Storage​

Fireship - Seven Database Paradigms​

Link: Seven Database Paradigms

An introduction to seven of the most common database paradigms and where they should be used. Short, sweet, and to the point.

CockroachDB - Architecture Resources and Documentation​

CockroachDB is a strongly consistent, distributed SQL database. Synnax takes quite a bit of inspiration from its high level architecture and its codebase. The following is a list of resources detailing CockroachDB's architecture, as well as some important RFCs that were used as inspiration for Synnax's design:

  • CockroachDB Architecture Overview
  • Video - CockroachDB, Architecture of a Geo-Distributed SQL Database
  • RFC - CockroachDB: Distributed SQL

Spanner - Google's Globally Distributed Database​

Spanner: Google’s Globally Distributed Database

Developed at Google to help power their search engine, Spanner is, in many ways, the mother of all strongly consistent distributed databases. This document summarizes Spanner's design and provides great background on many of the considerations that go into working with distributed systems.

Building Distributed Systems​

Chris Colohan - What is a Distributed System?​

Link: What is a Distributed System?

A ground introduction to what a distributed system is, why it exists, and some example of distributed systems in practice.

Robert Vitillo - Understanding Distributed Systems​

Link: Understanding Distributed Systems

An introduction to distributed systems. It leaves you with a good understanding of the high level components and rules. It covers the following:

  • Communication - Reliable Links, Secure Links, Discovery (DNS), and APIs
  • Coordination - System Models, Failure Detection, Clocks, Leader Election, Replication, Transactions
  • Scalability - Functional Decomposition, Partitioning, Duplication, Caching
  • Resiliency - Common Failure Causes, Downstream Resiliency, Upstream Resiliency
  • Testing - Continued Integration/Delivery, Monitoring, and Observability

Marten Van Steen - Distributed Systems​

Link: Distributed Systems

A deep dive into the fundamental concepts underlying the design of distributed systems. Contains info on the following categories:

  1. Architectural Styles
  2. Processes and Basic Concurrency Models
  3. Communication - Networking, Protocols, and Architectures
  4. Naming and Name Resolution
  5. Coordination - Clock Synchronization, Leader Election
  6. Consistency and Replication
  7. Fault Tolerance
  8. Security

Programming in Go​

FreeCodeCamp - Learn Go Programming​

Link: Learn Go Programming

A concise, practical introduction to Go. Not much else to say except that its a great way to get into the language.

C.A.R. Hoare - Communicating Sequential Processes​

Link: PDF - Communicating Sequential Processes

A deep dive into the theory and mathematics behind Go's concurrency model, Communicating Sequential Processes (CSP). It's pretty brutal and requires some knowledge of discrete mathematics, but it's a great read if you want to understand the mathematics behind writing concurrent programs.

Writing Great Software​

John K. Ousterhout - A Philosophy of Software Design​

Link: A Philosophy of Software Design

Discusses the fundamental problem in large scale software design: managing complexity. It focuses on patterns and principles for writing software that defines clear architectural boundaries, and is built to last.

Many Authors - Design Patterns: Elements of Reusable Object-Oriented Software​

Link: Design Patterns: Elements of Reusable Object-Oriented Software

The bible of software, unless you're a functional programmer. First published in 1994, this book is a collection of OOP design patterns that form the core of many of the most resilient software systems on the planet.
